Cómo funciona el módulo de Remitos
¿Qué es un remito?

Un remito es un comprobante interno que documenta la entrega de productos a un cliente, sin ser una factura. Es útil cuando:

  • El cliente se lleva la mercadería pero querés facturarla después (ej: mayoristas con cuenta mensual).
  • Querés llevar control de qué se entregó sin emitir factura electrónica en el momento.
  • Después de varios remitos podés consolidar todo en una sola factura.

El remito no se envía a ARCA ni tiene CAE. La factura posterior sí.

Numeración

Los remitos llevan el formato R 00001-00000001 (tipo + punto de venta + número correlativo). La numeración es automática y no se reutiliza.

¿Cuándo se descuenta el stock?

Al confeccionar el remito. Porque los productos ya salieron físicamente del depósito. Si después anulás el remito, el stock vuelve automáticamente.

Cuando después facturás el remito, la factura NO vuelve a descontar stock (ya fue descontado al hacer el remito).

Estados posibles
  • pendiente — recién creado, editable. Se puede modificar, anular o facturar.
  • liquidado — volvió el camión, se cargaron entregas/devoluciones/roturas. Listo para facturar por las cantidades reales.
  • facturado — se emitió una factura sobre este remito. Se puede ver pero no editar ni anular.
  • anulado — cancelado antes de facturar. El stock se devolvió. No se puede volver atrás.
Flujo típico
  1. Nuevo remito: seleccionás cliente, cargás productos, imprimís.
  2. El camión sale a hacer el reparto y entrega la mercadería.
  3. Cuando vuelve el camión: tocás el botón ⚖️ "Liquidar" en el remito. Cargás cuánto devolvió el cliente y cuánto se rompió en el viaje (si pasó algo). El remito queda en estado liquidado.
  4. Cuando el cliente quiere su factura: desde este listado tildás uno o varios remitos del mismo cliente → "Facturar seleccionados".
  5. Te lleva al módulo Nueva Venta con los items precargados (cantidades reales entregadas si fue liquidado, o cantidades originales si nunca se liquidó).
  6. Una vez emitida la factura, los remitos quedan en facturado con link a la factura.

💡 Si el cliente recibió todo conforme y no hubo devoluciones ni roturas, podés saltar el paso de liquidar y facturar directamente desde pendiente. La liquidación solo es necesaria cuando volvió mercadería o se rompió algo en el viaje.

Liquidación de remito (volvió el camión)

Es el paso opcional para ajustar el remito según lo que realmente recibió el cliente. Hay 3 columnas por cada producto:

  • Entregada — lo que el cliente se quedó. Se autocalcula a medida que cargás Devuelta y Rotura.
  • Devuelta — lo que el cliente rechazó/no quiso. Vuelve al stock automáticamente.
  • Rotura — lo que se rompió en el viaje (camión, manipulación, etc). NO vuelve al stock: queda registrado como pérdida (merma propia).

Casos típicos:

  • Todo entregado OK — Botón "Marcar todo como entregado" → confirmar.
  • Cliente rechazó un producto entero (ej: 10 papas, no quiso ninguna) → Devuelta=10, las papas vuelven a stock.
  • Cliente rechazó parcialmente (ej: pidió 20, se quedó con 10) → Devuelta=10, vuelven 10 a stock.
  • Se rompió en el viaje (caja se cayó del camión) → Rotura=lo que se rompió. No vuelve a stock, queda como merma.

📌 La suma de Entregada + Devuelta + Rotura debe ser igual a la cantidad original. El sistema te avisa si no cuadra (fila roja).
📌 Si el cliente rompe algo después de recibirlo, eso ya no es problema nuestro: dejalo como entregado y se le factura igual.
📌 Una vez liquidado, el remito no se puede anular. Si te equivocaste, hablá con el admin.

Facturar varios remitos juntos

Podés seleccionar múltiples remitos del mismo cliente (en estado pendiente o liquidado) y facturarlos en un solo comprobante. El sistema:

  • Consolida los items: si el mismo producto aparece en varios remitos, se suman las cantidades en una sola línea.
  • Usa los precios actuales del momento de facturar (no los del día en que se emitió cada remito).
  • Si alguno de los remitos está liquidado, usa las cantidades reales entregadas, no las originales.

¿Por qué precios actuales? Es la práctica estándar para mayoristas en Argentina. Con inflación, usar precios viejos es inviable administrativamente. El remito guarda los precios originales como referencia histórica.

Devoluciones después de facturar

Un remito ya facturado NO se puede anular (la factura electrónica ya está emitida).

Si el cliente devuelve algo después de que el remito fue facturado:

  1. Andá al módulo Notas de Crédito.
  2. Creá una nota de crédito asociada a la factura correspondiente.
  3. Especificá los items devueltos — el sistema reintegra stock y emite la nota de crédito electrónica en ARCA.

El remito original queda como está, la nota de crédito compensa la devolución.

Marcar manualmente como facturado

Esta opción (botón naranja en remitos pendientes) existe como red de emergencia: si emitiste la factura pero por algún problema (corte de luz, error de red) el sistema no marcó el remito automáticamente, podés asociarlo a mano ingresando el ID de la factura.

Usala solo cuando sea estrictamente necesario. Ingresar un ID incorrecto rompe la trazabilidad entre remito y factura.

Más cosas útiles
  • Los combos no se pueden remitir (tenés que cargar los productos base).
  • Los filtros aceptan combinaciones: podés ver remitos pendientes de un cliente específico en un rango de fechas.
  • El ícono imprime el remito en la impresora térmica.
