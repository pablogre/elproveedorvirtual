Cargar masivamente productos al sistema desde Excel/CSV. Casos de uso típicos:

  • Migración inicial al instalar el sistema (cargar los 500-2000 productos del comercio).
  • Lista nueva del proveedor: el proveedor te manda Excel con su catálogo y precios actualizados.
  • Carga de stock inicial al hacer un inventario.
  • Productos de temporada que se cargan en bloque (campañas, navidad, vuelta al cole).

Hacé backup antes de importar masivamente.

Probá primero con 10-20 productos. Si todo sale bien, repetís con el archivo completo. Una vez cargados, si algo está mal podemos borrarlos masivamente con un SQL — pero es mejor evitarlo.

Tango → Listados → Maestro de Artículos → Exportar a Excel.
Bejerman → Tablas → Artículos → Imprimir → Excel/CSV.
Excel propio del proveedor: muchos proveedores te pasan listas en Excel con código, descripción y precio. Andan perfecto.
PDF de catálogo: si solo tenés un PDF, contactanos. Se puede extraer con OCR pero requiere preparación previa.

Concepto clave: En este sistema, el costo se carga SIN IVA (neto). Es la base que usa el sistema para calcular los precios de las 5 listas mediante el margen.

Fórmula: Precio Lista N = Costo × (1 + Margen N / 100)
El IVA se aplica recién al momento de facturar (no está incluido en el costo ni en los precios de lista).
Caso A: el Excel SÍ trae costo

El sistema usa el costo del Excel y calcula automáticamente el margen real a partir del precio (lista 1) y ese costo. Ejemplo:

  • Excel trae: precio = $12.622,58, costo = $10.518,82
  • Sistema guarda: costo = $10.518,82, margen = 20% (calculado: (12622.58 - 10518.82) / 10518.82 × 100)
Caso B: el Excel NO trae costo

El sistema usa el "Margen default a aplicar" (lo configurás en la pantalla de importación, default 30%) y calcula el costo hacia atrás. Ejemplo:

  • Excel trae: precio = $12.622,58 (sin costo)
  • Margen default ingresado: 20%
  • Sistema guarda: costo = $10.518,82 (calculado: 12622.58 / 1.20), margen = 20%
Recomendación: si tu proveedor te pasa lista con precios netos y costo neto, podés importar directo. Si tu Excel viene con precios o costos con IVA incluido, primero dividilos por 1.21 antes de importar (o por 1.105 si son productos al 10.5%). Si querés ayuda con esa conversión avisanos, lo hacemos en una columna nueva del Excel y queda listo.

El sistema soporta hasta 5 precios distintos por producto (típicamente: minorista, mayorista, distribuidor, mayorista grande, especial). Cada cliente está asignado a una lista, y al facturarle se usa el precio de esa lista.

En el Excel podés tener:
  • Solo una columna "Precio" → va a Lista 1. Las otras 4 listas quedan vacías. Lo más común.
  • Cinco columnas "Precio 1", "Precio 2", "Precio 3", "Precio 4", "Precio 5" → se mapean directo a las 5 listas.
  • Variantes: "Precio Lista 1", "P1", "Lista 1" — todas se reconocen.
Después de importar:

Si solo cargaste lista 1, podés generar las otras listas con un margen desde el ABM de productos (botón "Recalcular precios por margen"). O completarlas a mano producto por producto.

Recordá: los 5 precios se cargan sin IVA (igual que el costo). El IVA se aplica recién al facturar, según la alícuota del producto (21% por default).

La categoría es texto libre en el modelo (no es FK a otra tabla). Esto significa que:

  • Podés cargar cualquier texto como categoría: "Almacén", "Bebidas Sin Alcohol", "Limpieza", etc.
  • Si la categoría no existe, se "crea" automáticamente al asignársela al producto.
  • Si una categoría se escribe distinto en filas distintas (ej: "Bebidas" vs. "BEBIDAS" vs. "Bebida"), el sistema las trata como categorías distintas.
Tip: antes de importar, abrí el Excel y unificá la columna Categoría (sacá variaciones de mayúsculas, plurales, etc.). Eso te ahorra dolores de cabeza después.

Pesables (carnicería, fiambrería, verdulería)

Si tenés productos que se venden por peso, agregá una columna "Pesable" al Excel con valores S/N (también acepta Si/No, 1/0, true/false). Los marcados como pesables se procesan distinto en la facturación (el sistema pide el peso al venderlos).

Código de barras

Columna opcional. Aceptamos cualquier formato (EAN-13, código interno de 5 dígitos para balanza, etc.). El sistema NO valida el dígito verificador del EAN — lo carga tal cual viene.

Stock mínimo

Si querés cargar el umbral de stock mínimo (para que el sistema te alerte cuando bajás de ese valor), agregá la columna "Stock Mínimo". Si no, queda en 0.

Si tu Excel tiene columnas con nombres raros, en el paso 2 las mapeás manualmente con los dropdowns.

El sistema identifica los duplicados por código:

  • Si el código ya existe en la BD → la fila se saltea.
  • Las filas saltadas figuran en el reporte final con su número de fila y código.
¿Querés actualizar precios masivamente? El importador actual solo crea productos nuevos. Para actualizar precios de productos existentes (típico cuando el proveedor te manda lista nueva), tenemos que armar un módulo aparte. Contactanos.

Código:
  • Obligatorio. Hasta 50 caracteres. Debe ser único.
Nombre:
  • Obligatorio. Hasta 200 caracteres.
Precios y costo:
  • Acepta números con coma o punto decimal ("1.234,56" → 1234.56). Acepta símbolo $ y se ignora.
  • Si el precio (lista 1) está vacío o es 0 → fila descartada.
  • Los precios 2-5 son opcionales.
IVA:
  • Acepta valores entre 0 y 50 (porcentaje).
  • Si está vacío o fuera de rango, se asigna 21% por default.
Stock:
  • Acepta decimales (para pesables: 1.500 kg).
  • Si está vacío, se asigna 0.

❓ "Falta código" en muchas filas
El detector mapeó mal la columna Código. Andá al paso 2 y verificá. A veces lo confunde con el código de barras o con el nombre.
❓ "Precio inválido"
La columna Precio tiene valores no numéricos (texto, vacío, "consultar", etc.). Limpiá el Excel: solo números o vacío. El sistema acepta formato argentino "1.234,56" sin problema.
❓ Importé pero los precios no son correctos
Verificá si tu Excel viene con precios con IVA incluido o sin IVA (netos). El sistema espera precios SIN IVA (netos), igual que el costo. Si tu Excel viene con IVA incluido, dividí los precios por 1.21 antes de importar (o por 1.105 para productos al 10.5%).
❓ Todos los márgenes quedaron iguales (default)
El Excel no traía costo, así que el sistema usó el margen default. Si querés márgenes reales, agregá la columna "Costo" al Excel y volvé a importar (después de borrar los importados anteriores).
❓ Las categorías quedaron desprolijas
Las categorías se cargan tal cual vienen en el Excel. Si tenés "BEBIDAS" y "Bebidas" como categorías distintas, hay que ir al ABM y unificar. La próxima vez, antes de importar, unificá la columna en el Excel.
